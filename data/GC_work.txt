Объект USer
Поля : int age, String name
Размер User в памяти: 16 байт(заголовок) + 8 байт(размер int с учетом выравнивания) + 48 байт (размер строки с минимум 1 символом).
Итого : 72 байта.

Задаем размер heap с помощью ключей Xmx и Xms.
1. Все новые объекты (new User) попадают в область heap под названием eden space.
2. Как только eden заполняется вызывается minor GC, проверяющий существование ссылок у объектов в eden. У кого ссылок не обнаружено, тот удаляется, остальные направляются в survivor space.
3. Объекты, пережившие несколько перемещений в survivor space, получают promotion и перемещаются в следующую область heap, называюмую Old(tenured) generation.
4. Когда Old generation заполняется, вызывается major GC , затрагивающий весь old generation heap.
